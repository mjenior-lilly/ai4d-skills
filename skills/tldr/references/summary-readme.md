# Repository README Summary

Role: expert at analyzing and summarizing software packages, libraries, or
repositories for scientists and developers evaluating whether the tool meets
their needs.

Extract these elements before writing:
- Purpose and scope — what the tool does, what problems it solves
- Core features and functionality — main capabilities or modules
- Installation and setup
- Typical usage patterns, including example commands or workflows if provided
- Dependencies and requirements — languages, platforms, libraries
- Limitations or known issues
- Project status and maintenance — activity level, support, contribution
  guidelines, future plans

## Output format

Use exactly these sections:

**1. Package Name and Purpose** — name of the package/repository and a
one-sentence summary of its primary purpose.

**2. Core Functionality** — bullet list or short paragraph of key features and
capabilities.

**3. Installation and Setup** — installation steps, package managers used,
system requirements.

**4. Usage Overview** — typical usage patterns with examples, if available.

**5. Dependencies and Requirements** — languages, platforms, libraries, or
tools required.

**6. Limitations and Known Issues** — limitations, performance constraints,
unsupported use cases, known bugs.

**7. Project Status and Maintenance** — activity level (actively maintained,
archived), contributor guidance, links to issues, roadmap mentions.

Write in clear, professional technical language for an internal R&D or software
engineering audience. Do not add external evaluations or interpretation beyond
what the README states. Apply the shared guardrails from SKILL.md.
